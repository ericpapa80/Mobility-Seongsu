# PRD: 다중 출처 스크래이핑 시스템

## 1. 프로젝트 개요

### 1.1 목적
다양한 출처의 웹사이트로부터 데이터를 체계적으로 수집하고, JSON 및 CSV 형식으로 저장하는 확장 가능한 스크래이핑 시스템을 구축합니다.

### 1.2 범위
- SGIS (통계지리정보서비스) 데이터 수집
- 다양한 출처의 스크래이퍼 추가 가능한 확장 가능한 아키텍처
- JSON 및 CSV 형식의 데이터 저장
- 환경 변수를 통한 인증 정보 관리

## 2. 요구사항 정의

### 2.1 기능 요구사항

#### FR1: SGIS 데이터 수집
- **설명**: SGIS API를 통해 POI 회사 밀도 데이터를 수집
- **우선순위**: 높음
- **입력**: API 파라미터 (선택사항)
- **출력**: JSON 및 CSV 형식의 데이터 파일

#### FR2: 다중 포맷 저장
- **설명**: 수집된 데이터를 JSON과 CSV 형식으로 저장
- **우선순위**: 높음
- **특징**: 
  - 타임스탬프 기반 파일명 생성
  - 중첩된 데이터 구조의 평탄화 지원
  - 다중 라인 필드 값의 단일 라인 변환

#### FR3: 인증 관리
- **설명**: .env 파일을 통한 안전한 인증 정보 관리
- **우선순위**: 높음
- **요구사항**:
  - AccessToken 및 JSESSIONID 관리
  - 환경 변수 검증

#### FR4: 로깅 시스템
- **설명**: 구조화된 로깅 시스템
- **우선순위**: 중간
- **요구사항**:
  - 파일 및 콘솔 출력
  - 로그 레벨 관리
  - 타임스탬프 기반 로그 파일

#### FR5: 확장 가능한 아키텍처
- **설명**: 새로운 스크래이퍼 추가 용이
- **우선순위**: 높음
- **요구사항**:
  - BaseScraper 추상 클래스
  - 공통 유틸리티 모듈
  - 표준화된 인터페이스

### 2.2 비기능 요구사항

#### NFR1: 유지보수성
- 모듈화된 코드 구조
- 명확한 폴더 구조
- 문서화

#### NFR2: 확장성
- 새로운 스크래이퍼 추가 시 최소한의 코드 수정
- 공통 기능의 재사용

#### NFR3: 안정성
- 에러 핸들링
- 재시도 로직
- 데이터 검증

## 3. 시스템 아키텍처

### 3.1 폴더 구조

```
all_scrapping/
├── config/                 # 설정 관리
│   └── settings.py         # 환경 변수 로드 및 설정
├── scrapers/               # 스크래이퍼 모듈
│   ├── base.py             # 기본 스크래이퍼 클래스
│   └── sgis/               # SGIS 전용 스크래이퍼
│       ├── sgis_scraper.py
│       └── api_client.py
├── utils/                  # 공통 유틸리티
│   ├── file_handler.py    # 파일 저장 (JSON/CSV)
│   ├── auth_manager.py    # 인증 관리
│   └── logger.py          # 로깅
├── data/                   # 수집된 데이터
│   ├── raw/               # 원본 데이터
│   └── processed/         # 가공된 데이터
├── logs/                   # 로그 파일
└── scripts/                # 실행 스크립트
    └── run_sgis.py
```

### 3.2 컴포넌트 설계

#### 3.2.1 BaseScraper
- 모든 스크래이퍼의 기본 클래스
- 공통 메서드: `fetch()`, `save()`, `validate()`
- 추상 메서드: `scrape()` (각 스크래이퍼에서 구현)

#### 3.2.2 SGISScraper
- BaseScraper 상속
- SGIS API를 통한 데이터 수집
- JSON/CSV 저장 기능

#### 3.2.3 SGISAPIClient
- SGIS API 호출 전용 클라이언트
- 세션 관리 및 쿠키 처리
- 재시도 로직

#### 3.2.4 FileHandler
- JSON/CSV 저장 기능
- 데이터 포맷팅 및 검증

#### 3.2.5 AuthManager
- .env 파일에서 인증 정보 로드
- 인증 정보 검증

#### 3.2.6 Settings
- 환경 변수 관리
- 설정 검증

## 4. 데이터 수집 프로세스

### 4.1 SGIS 데이터 수집 프로세스

1. **초기화**
   - Settings 로드
   - AuthManager를 통한 인증 정보 검증
   - SGISAPIClient 초기화

2. **데이터 수집**
   - API 엔드포인트 호출: `getPoiCompanyDensity.json`
   - 요청 헤더 및 쿠키 설정
   - 응답 데이터 수신

3. **데이터 검증**
   - 데이터 형식 검증
   - 필수 필드 확인

4. **데이터 저장**
   - 타임스탬프 기반 디렉토리 생성
   - JSON 형식 저장
   - CSV 형식 저장 (데이터 평탄화)

5. **로깅**
   - 수집 과정 로깅
   - 에러 발생 시 에러 로깅

## 5. 확장 계획
