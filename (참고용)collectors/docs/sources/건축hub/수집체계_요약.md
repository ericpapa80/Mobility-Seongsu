# 건축Hub OpenAPI 수집 체계 요약

`collectors/docs/sources/건축hub` 문서와 OpenAPI 활용가이드를 기준으로 **건축물대장·건축인허가·주택인허가** 데이터 수집 방식을 정리한 요약입니다.

---

## 1. 인증 및 .env

- **공공데이터포털** OpenAPI와 동일한 방식의 **서비스 Key** 사용.
- 요청 시 쿼리 파라미터: `serviceKey=인증키`
- **.env 예시 (env.sample 29~30줄 형식)**

  ```env
  # 건축Hub API
  건축Hub_API_KEY=d24a839e1....
  ```

- 코드에서 사용 시: `건축Hub_API_KEY` 값을 읽어 **요청 파라미터 `serviceKey`** 에 넣어 사용하면 됨.

---

## 2. 건축물대장 (BldRgstHubService)

| 항목 | 내용 |
|------|------|
| **서비스 ID** | BldRgstHubService |
| **Base URL** | `https://apis.data.go.kr/1613000/BldRgstHubService` |
| **인증** | 쿼리 파라미터 `serviceKey` (공공데이터포털 발급 키) |
| **응답 형식** | `_type=json` 또는 XML |
| **페이징** | `numOfRows` 최대 **100건**, `pageNo`로 페이지 번호. 응답 `totalCount`로 전체 건수 확인 후 반복 요청 |

### 필수 요청 파라미터 (공통)

- **sigunguCd**: 시군구코드 (행정표준코드, 5자리)  
  - 예: 성동구 = **11200**
- **bjdongCd**: 법정동코드 (행정표준코드)  
  - 행정표준코드관리시스템 https://www.code.go.kr → 코드검색 → 법정동코드목록조회에서 조회

### 옵션 파라미터

- platGbCd: 대지구분코드 (0:대지, 1:산, 2:블록)
- bun, ji: 번·지
- startDate, endDate: 검색시작일·검색종료일 (YYYYMMDD)
- numOfRows, pageNo: 페이징

### 오퍼레이션 목록 (건축물대장)

| 번호 | 오퍼레이션(영문) | 국문 |
|------|-------------------|------|
| 1 | getBrBasisOulnInfo | 건축물대장 기본개요 조회 |
| 2 | getBrRecapTitleInfo | 건축물대장 총괄표제부 조회 |
| 3 | getBrTitleInfo | 건축물대장 표제부 조회 |
| 4 | getBrFlrOulnInfo | 건축물대장 층별개요 조회 |
| 5 | getBrAtchJibunInfo | 건축물대장 부속지번 조회 |
| 6 | getBrExposPubuseAreaInfo | 건축물대장 전유공용면적 조회 |
| 7 | getBrWclfInfo | 건축물대장 오수정화시설 조회 |
| 8 | getBrHsprcInfo | 건축물대장 주택가격 조회 |
| 9 | getBrExposInfo | 건축물대장 전유부 조회 |
| 10 | getBrJijiguInfo | 건축물대장 지역지구구역 조회 |

### 요청 예시 (표제부 조회)

```
GET https://apis.data.go.kr/1613000/BldRgstHubService/getBrTitleInfo
  ?sigunguCd=11200
  &bjdongCd=10300
  &platGbCd=0
  &bun=0012
  &ji=0000
  &numOfRows=10
  &pageNo=1
  &_type=json
  &serviceKey={건축Hub_API_KEY}
```

### 응답 구조 (JSON)

```json
{
  "response": {
    "header": { "resultCode": "00", "resultMsg": "NORMAL SERVICE" },
    "body": {
      "items": { "item": { ... } },
      "numOfRows": 10,
      "pageNo": 1,
      "totalCount": 23
    }
  }
}
```

## 5. 수집 체계 정리

### 단위

- **시군구코드(sigunguCd) + 법정동코드(bjdongCd)** 단위로 API 호출.
- 동 단위로 한 번에 조회 후, **totalCount** 확인 후 **numOfRows=100, pageNo=1,2,...** 로 페이징.

### 코드 참고

- **시군구코드**: `첨부2. 시군구코드에 따른 통합분류코드 목록.pdf`  
  - 예: 서울 성동구 = **11200**
- **법정동코드**: 행정표준코드관리시스템(code.go.kr) 법정동코드목록조회에서 조회 (가이드 1.3 참고사항).

### 수집 흐름 (구현 시 참고)

1. 대상 지역의 시군구코드·법정동코드 목록 확보 (예: 성동구 11200 + 성수동 등 법정동 코드).
2. 건축물대장: `BldRgstHubService` + 필요한 오퍼레이션별로  
   `sigunguCd`, `bjdongCd` 고정, `numOfRows=100`, `pageNo` 증가시키며 반복 요청.  
   응답 `totalCount`로 총 페이지 수 계산.
3. 건축인허가: `ArchPmsHubService` 동일 방식으로 시군구/법정동·페이징 처리.
4. 인증: 모든 요청에 `.env`의 `건축Hub_API_KEY` 값을 `serviceKey`로 포함.

### 제한 사항

- 1회 요청 최대 **100건** (numOfRows).
- 응답에 totalCount가 있으므로, 첫 요청으로 전체 건수 확인 후 페이지 반복 요청하면 됨.
