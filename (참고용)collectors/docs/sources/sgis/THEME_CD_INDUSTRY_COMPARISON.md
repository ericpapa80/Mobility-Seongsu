- **용도**: 표준산업분류 체계에 따른 상세한 업종 분류

### 4. 매핑 가능성 평가

#### ✅ 가능한 부분

1. **대략적인 업종 추정**: 회사명 키워드 분석을 통해 theme_cd별로 주요 업종을 추정할 수 있음
2. **대분류 수준 매핑**: 일부 theme_cd는 특정 대분류 업종에 집중되어 있음
   - 예: theme_cd=220 → 정보통신업(J) 관련
   - 예: theme_cd=231 → 전문서비스업(M) 관련
   - 예: theme_cd=235 → 건설업(F), 전문서비스업(M) 관련

#### ❌ 제한적인 부분

1. **정확한 매핑 불가**: theme_cd 하나가 여러 업종구분코드를 포함할 수 있음
2. **공식 매핑 테이블 부재**: SGIS 공식 문서에서 theme_cd와 업종구분코드의 직접적인 매핑 테이블을 찾을 수 없음
3. **회사명 기반 추정의 한계**: 회사명만으로는 정확한 업종을 판단하기 어려움

### 5. 결론

#### theme_cd로 업종 식별 가능 여부

**부분적으로 가능하나, 정확도에 한계가 있음**

1. **대분류 수준**: theme_cd를 통해 업종을 대분류 수준으로 식별하는 것은 가능
   - 예: theme_cd=220 → 정보통신업(J) 관련
   - 예: theme_cd=231 → 전문서비스업(M) 관련

2. **상세 분류**: 세부 업종구분코드(Level 3~5)까지 정확히 식별하기는 어려움

3. **정확도 향상 방법**:
   - SGIS 개발자 지원센터에서 공식 매핑 테이블 확인
   - 각 회사의 사업자등록번호를 통해 실제 업종 확인
   - 회사명 외에 추가 정보(주소, 업종명 등) 활용

### 6. SGIS 개발자 지원센터 확인 결과

**확인 일자**: 2025-11-30

**확인 내용**:
1. SGIS 개발자 지원센터(https://sgis.mods.go.kr/developer/html/newOpenApi/api/dataApi/) 접속 확인
2. 기술업종 통계 API 문서 확인
   - API에서 `techbiz_cd`(기술업종코드) 사용 확인
   - 예: "첨단기술"(techbiz_cd: "11"), "고기술"(techbiz_cd: "12")
3. **공식 매핑 테이블 미제공**: 
   - theme_cd와 업종구분코드 간의 직접적인 매핑 테이블은 공식 문서에서 제공되지 않음
   - theme_cd와 techbiz_cd 간의 매핑 정보도 명시적으로 제공되지 않음
   - API 문서에는 "기술업종 코드표 참고"라고만 언급되어 있으나, 구체적인 코드표는 제공되지 않음

**결론**: 
- SGIS 공식 문서에서는 theme_cd와 표준산업분류 업종구분코드 간의 직접적인 매핑 테이블을 제공하지 않음
- theme_cd는 SGIS 내부에서 사용하는 기술업종 분류 코드로 보이며, 표준산업분류와는 별개의 체계로 운영되는 것으로 추정됨
