# fusionstats.json 요청 파라미터 범주별 의미

결합조건 API(`ServiceAPI/stats/fusionstats.json`) 요청 시 사용하는 **params** 범주별 의미와 값 설명입니다.  
수집 스크립트: `scripts/sgis_fusion/run_fusion_pop_house_household.py`  
참고: `Combined_Pop-Family-Housing.txt`, [SGIS 개발자센터 센서스 통계](https://sgis.mods.go.kr/developer/html/newOpenApi/api/dataApi/census.html)

---

## 1. 결합 기준·쿼리 타입

| 파라미터 | 의미 | 예시 값 | 비고 |
|----------|------|---------|------|
| **combine_base** | 결합 조건의 기준이 되는 통계 | `population` | 인구를 기준으로 결합할 때 사용. |
| **fusion_query_type** | 조회할 결합 유형 | `population_household_house` | 인구·가구·주택 세 가지를 동시에 만족하는 건수 조회. |

---

## 2. 인구(연령) 조건

| 파라미터 | 의미 | 예시 값 | 비고 |
|----------|------|---------|------|
| **age_from** | 연령 하한(세) | `20` | 0~150. 해당 세 이상만 포함. |
| **age_to** | 연령 상한(세) | `39` | 0~150. 해당 세 이하만 포함. (age_from ≤ 나이 ≤ age_to) |

공식 OpenAPI의 `householdmember.json`에서도 동일하게 `age_from`, `age_to`(0~150)를 사용합니다.  
**나이·연도별 수집**: 동일 API로 연령대(age_from/age_to)와 기준 연도(year)를 바꿔가며 수집 가능하다. 예: 20~39세/2024년, 40~59세/2023년 등. `Combined_Pop-Family-Housing.txt` 참고.

### 수집 가능한 연령 구분 기준

| 구분 | API | 연령 기준 | 비고 |
|------|-----|-----------|------|
| **fusionstats (결합조건)** | ServiceAPI/stats/fusionstats.json | **고정 코드 없음**. `age_from`, `age_to`에 0~150 사이 값을 넣어 **임의 구간** 지정. | 예: 20~39세 → age_from=20, age_to=39. 10대 → 10, 19. 전 연령 → 0, 150. |
| **공식 OpenAPI 인구통계** | OpenAPI3/stats/searchpopulation.json | **age_type** 연령타입코드표 사용. 개발자센터 인구통계 API에서 “연령타입코드표 참고” 버튼으로 코드표 확인. | 결합조건 API와는 별도. 인구 단일 조회용. |

**통계청·인구주택총조사에서 쓰는 5세 단위 연령 구간** (참고용, fusionstats에서 age_from/age_to로 동일 구간 지정 가능):

| 10~14세 | 10 | 14 | |
| 15~19세 | 15 | 19 | |

| 25~29세 | 25 | 29 | |
| 30~34세 | 30 | 34 | |
| 35~39세 | 35 | 39 | |

10년 단위(20~29세, 30~39세 등)나 경제활동인구 구간(15~64세) 등은 위 구간을 조합해 `age_from`/`age_to`로 설정하면 됩니다.

## 4. 주택 조건

| 파라미터 | 의미 | 예시 값 | 비고 |
|----------|------|---------|------|
| **house_area_cd** | 주택면적 구분 | `01,02` 또는 `01,02,...,08` | 주택면적 코드. 다중 선택 시 쉼표 구분. "0㎡ 초과 ~ 230㎡ 이하" 선택 시 01~08만 전송(09 제외). [HouseAreaCode](https://sgis.mods.go.kr/developer/html/openApi/api/dataCode/HouseAreaCode.html) 참고. |
| **house_type** | 주택유형 구분 | `02,03,04,05` 등 | 대화형통계지도 UI에서 주택유형 다중 선택 시 **쉼표 구분**으로 전송. 01=단독주택, 02=아파트, 03=연립, 04=다세대, 05=비거주용건물내주택, 06=주택이외의거처. [HouseCode](https://sgis.kostat.go.kr/developer/html/openApi/api/dataCode/HouseCode.html) 참고. |

주택면적 코드는 전용면적 구간별 코드이며(01: 20㎡ 이하, 02: 20~40㎡, 03: 40~60㎡ 등).  
주택**유형**(단독주택·아파트 등)과 주택**면적**은 별도 항목이며, UI에서 둘 다 다중 선택 시 payload에는 각각 쉼표 구분 문자열로 나갑니다.

**연면적이 전용면적인 근거**: 인구주택총조사 규칙에는 "주거용 연면적"만 규정되어 있고 전용/공급 용어는 없음. 등록센서스에서는 주거용 연면적을 **건축물대장** 등 행정자료로 대체하여 조사함. 건축물대장에서 집합건축물(아파트 등)은 전유부에 **전용면적**·공용면적을 구분 기재하고, 일반건축물(단독주택 등)은 전유부가 없어 기재 면적을 전용면적으로 해석함(건축물대장의 기재 및 관리 등에 관한 규칙). 따라서 세대 단위로 나오는 주거용 연면적은 **전용면적**에 대응한다고 보는 해석이 타당함. 공식 문구는 통계청·KOSIS·SGIS 메타데이터에서 재확인 권장.

## 5. 연도·경계

| 파라미터 | 의미 | 예시 값 | 비고 |
|----------|------|---------|------|
| **year** | 통계 기준 연도 | `2024` | 인구/주택: 2015~2024 등. **연도별 수집** 시 이 값만 바꿔 반복 요청하면 됨. |
| **bnd_year** | 집계구 경계 기준 연도 | `2025` | 결과가 나오는 집계구 경계의 연도. |

---

## 6. 영역·단계

| 파라미터 | 의미 | 예시 값 | 비고 |
|----------|------|---------|------|
| **adm_cd** | 행정구역 코드 | `11040680` | 8자리: 읍면동 단위. 요청당 하나, 스크립트는 동 단위로 반복 요청. |
| **low_search** | 하위 통계 단계 | `0`, `1`, `2` | 0: 해당 구역만. 1: 1단계 하위(예: 읍면동 내 집계구). 2: 2단계 하위. 집계구 단위 결과를 받을 때는 `1` 사용. (소수 보호·비공개 처리와는 별도.) |
| **area_type** | 영역 타입 | `0` | 대화형통계지도에서 사용하는 영역 구분. 기본 0 사용. |

### adm_cd 길이 (공식 API와 동일 체계)

- `non`: 전국 시도 목록  
- 2자리: 시도  
- 5자리: 시군구  
- 8자리: 읍면동  
fusionstats에서는 보통 **8자리(읍면동)** 로 요청하면 해당 동 내 집계구 단위 결과가 반환됩니다.

## 7. 요약: 기본 예시 조합의 의미

`request_params.json`에 자주 쓰는 조합 예:

```json
{
  "combine_base": "population",
  "age_from": 20,
  "age_to": 39,
  "household_type": "A0",
  "house_area_cd": "01,02",
  "area_type": "0",
  "year": 2024,
  "fusion_query_type": "population_household_house",
  "low_search": 1,
  "bnd_year": 2025
}
```

| 범주 | 의미 |
|------|------|
| 인구 | 만 20세 이상 39세 이하 |
| 가구 | 1인가구(A0) |
| 주택 | 주택면적 코드 01 또는 02에 해당하는 면적 구간 |
| 연도 | 2024년 통계, 2025년 경계 |
| 영역 | 읍면동(adm_cd)별 요청, 1단계 하위(집계구)까지 |

응답의 `data_cnt`는 **해당 집계구(adm_cd) 안에서 위 인구·가구·주택 조건을 동시에 만족하는 건수**입니다. (일부 집계구는 비공개로 `N/A`.)

## 8. 실측 payload (대화형통계지도 F12 캡처)

결합 호출 조건에 따른 숫자 합계 차이 확인을 위해, 대화형통계지도 UI에서 체크박스 선택 시 F12 개발자도구로 캡처한 **fusionstats.json** Form Data 예시를 정리합니다.

### UI 선택 → payload 매핑 (캡처 기준)

| UI 항목 | 선택 예시 | payload 키 | payload 값 예시 |
|--------|------------|------------|------------------|
| 조사년도 | 2024년도 | year | 2024 |
| 기준 | 인구(명) 기준 | combine_base | population |
| 연령 | 20세 이상 ~ 30세 미만 | age_from, age_to | 20, 29 |
| 세대구성 | 1인가구만 체크 | household_type | A0 |
| 주택유형 | 아파트·연립·다세대·비거주용건물내주택 체크 (단독주택·주택이외의거처 미체크) | house_type | 02,03,04,05 (또는 02,03,04,05,06) ※ |
| 연면적 | 0㎡ 초과 ~ 230㎡ 이하 | house_area_cd | 01,02,03,04,05,06,07,08 |
| (고정) | — | area_type | 0 |
| (고정) | — | fusion_query_type | population_household_house |
| (고정) | 집계구 단위 | low_search | 1 |
| (고정) | — | adm_cd, bnd_year | 예: 11040680, 2025 |

※ 캡처 시 "주택이외의 거처"는 미체크였으나 payload에 06이 포함된 사례가 있었음. 재현 과정에서의 선택 실수일 수 있음.

### payload 특성 정리

- **house_type**, **house_area_cd**: UI에서 다중 선택 시 **쉼표 구분 문자열**로 한 번에 전송됨. (예: `house_type=02,03,04,05,06`, `house_area_cd=01,02,...,08`)
- **low_search=1**: 읍면동(adm_cd) 요청 시 1단계 하위(집계구)까지 결과 반환. 소수 보호(비공개)는 응답의 `N/A` 처리로 이루어지며, 이 파라미터와 직접 대응되지 않음.
- 스크립트에서 **ht/ha 셀별 합계**를 만들려면 (연령×주택유형×주택면적) 조합마다 **단일 값**으로 요청해 셀 단위 건수를 받고 합산하는 방식이 필요함. UI처럼 한 번에 다중 코드를 보내면 해당 조건을 **합친** 건수만 반환됨.
