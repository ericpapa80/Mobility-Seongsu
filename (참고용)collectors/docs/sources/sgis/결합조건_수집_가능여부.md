# SGIS 결합조건(인구·가구·주택) 수집 가능 여부

`Combined_Pop-Family-Housing.txt`의 결합조건과 `SGIS_OpenAPI_정의서.pdf`(및 공식 개발자센터 API 문서)를 비교한 결론입니다.

---

## 1. 결합조건 요약 (Combined_Pop-Family-Housing.txt)

- **대상**: 대화형통계지도 – 인구주택총조사 조건설정(결합조건)
- **엔드포인트**: `https://sgis.mods.go.kr/ServiceAPI/stats/fusionstats.json` (POST, form body)
- **조건 예시**:
  - `combine_base=population`
  - `age_from=20`, `age_to=39` (연령)
  - `household_type=A0` (1인가구)
  - `house_area_cd=01,02` (주택면적)
  - `fusion_query_type=population_household_house`
  - `adm_cd=11040680`, `low_search=1`, `year=2024`, `bnd_year=2025`
- **응답**: 집계구(또는 하위)별 `adm_cd`, `data_cnt` (동일 집계구 내에서 **인구·가구·주택 조건을 동시에 만족하는 건수**)
- **인증**: Cookie (`JSESSIONID`, `accessToken`), Referer `populationHouseView` 등 (포털 세션)

각 파라미터(연령, 세대구성, 주택면적, 연도, 영역 등)의 범주별 의미와 코드표는 **`fusionstats_params_guide.md`** 에 정리되어 있다.

---

## 2. 공식 OpenAPI 정의서/개발자센터 내용

개발자센터 [센서스 통계](https://sgis.mods.go.kr/developer/html/newOpenApi/api/dataApi/census.html) 기준:

| 구분 | 요청주소 | 결합 여부 | 비고 |
|------|----------|-----------|------|
| 총조사 주요지표 | `OpenAPI3/stats/population.json` | 단일 | tot_ppltn, tot_family, tot_house 등 종합 지표 |
| **인구통계** | `OpenAPI3/stats/searchpopulation.json` | 단일 | `age_type`, gender, adm_cd, low_search |
| **가구통계** | `OpenAPI3/stats/household.json` | 단일 | `household_type`(예: A0), adm_cd, low_search |
| **주택통계** | `OpenAPI3/stats/house.json` | 단일 | `house_type`, `house_area_cd`, const_year, adm_cd, low_search |

- **fusionstats / 인구·가구·주택 동시 결합 API**는 개발자센터·OpenAPI 정의서에 **기재되어 있지 않음**.
- 공식 문서에는 인구·가구·주택이 **각각 별도 API**로만 제공됨.

---

## 3. 결합조건 수집 가능 여부 정리

### 3.1 “결합조건과 동일한 지표” (한 집계구에서 세 조건 **동시 만족** 건수)

- **공식 OpenAPI3만으로는 불가**
  - 동일 `adm_cd`에 대해 “20~39세 **이면서** 1인가구 **이면서** 주택면적 01·02”인 **건수**를 반환하는 API는 정의서/개발자센터에 없음.
  - 인구·가구·주택을 각각 따로 조회하면, 같은 집계구라도 “동시 만족” 건수는 계산할 수 없음(집계 단위가 같아도 서로 다른 조회 결과의 교집합 수치를 서버가 제공하지 않음).

### 3.2 ServiceAPI fusionstats 사용 시

- **기술적으로는 수집 가능**
  - `Combined_Pop-Family-Housing.txt`에 기록된 대로 요청 시 집계구별 `data_cnt`를 받을 수 있음.
- **제한·리스크**
  1. **인증**: OpenAPI3의 `consumer_key`/`consumer_secret` 방식이 아니라, **포털 세션 쿠키**(`accessToken`, `JSESSIONID`)에 의존. 동일 accessToken 호환 여부는 문서에 없어 별도 확인 필요.
  2. **비공개 API**: 개발자센터·정의서에 없으므로 스펙 변경·중단 가능성 있고, 이용약관상 자동 수집 허용 여부도 불명.
  3. **호스트**: `sgis.mods.go.kr/ServiceAPI`는 대화형통계지도용 백엔드로 보이며, 공식 OpenAPI 호스트(`sgisapi.kostat.go.kr` 또는 `sgisapi.mods.go.kr/OpenAPI3`)와 다름.

## 4. 권장 방향

| 목표 | 권장 |
|------|------|
| “결합조건과 완전 동일”한 지표(동시 만족 건수)를 **공식·안정적으로** 수집 | **불가**. 해당 지표는 현재 공식 OpenAPI에 없음. |
| 결합조건과 **유사한** 분석용 데이터 수집 | **가능**. OpenAPI3만 사용해 동일 `adm_cd`에 대해 인구(연령별)·가구(1인가구)·주택(면적별)을 **각각** 조회 후, 집계구 경계와 함께 병렬 속성으로 붙여 사용. (예: 기존 `run_sgis_seongsu_tract_boundary_1person_household.py`처럼 경계 + 통계 병합) |
| fusionstats와 동일한 수치가 **반드시** 필요하고, 단기 스크립트/검증용으로만 사용 | 기술적으로 fusionstats 호출은 가능하나, **비공개·세션 의존**을 전제로 하며, 장기 사용·재현성·정책 리스크는 사용 주체가 검토해야 함. |

---

## 5. 참고

- **OpenAPI3 인증**: `auth/authentication.json` → `consumer_key`, `consumer_secret`으로 `accessToken` 발급. (기존 스크립트는 `sgisapi.kostat.go.kr/OpenAPI3` 사용)
- **집계구 경계**: `OpenAPI3/boundary/statsarea.geojson` (정의서 36번 집계구경계)
- **연령**: `searchpopulation.json`의 `age_type` 코드표는 개발자센터 “연령타입코드표 참고”로 확인 필요. 20~39세 구간에 해당하는 `age_type`이 있으면, 결합이 아닌 “연령별 인구” 수집은 OpenAPI3로 가능.

---

## 6. 수집 코드·데이터 폴더 (비공개 전제 구현 시)

비공개·세션 의존을 전제로 수집 스크립트를 둔 경우, 기존 SGIS와 구분하기 위해 다음처럼 분리해 두었습니다.

| 구분 | 기존 SGIS (공식 OpenAPI3) | 결합조건 수집 (비공개) |
|------|---------------------------|------------------------|
| 스크립트 | `collectors/scripts/sgis/` | `collectors/scripts/sgis_fusion/` |
| 수집 데이터 | `collectors/data/raw/sgis/` | `collectors/data/raw/sgis_fusion/` |

- 스크립트: `scripts/sgis_fusion/run_fusion_pop_house_household.py`
- 환경변수: `SGIS_FUSION_ACCESS_TOKEN` 또는 `SGIS_FUSION_COOKIE`
- 설명: `scripts/sgis_fusion/README.md` 참고

---

**요약**: 결합조건(인구+가구+주택 동시 만족)으로 수집하는 것은 **공식 OpenAPI 정의서 기준으로는 제공되지 않으며**, 동일 지표는 **fusionstats(비공개·세션 의존)** 를 쓸 때만 기술적으로 가능합니다. 공식만 사용할 경우 동일 행정/집계구에 대해 인구·가구·주택을 각각 조회해 병렬로 결합하는 방식만 가능합니다.
