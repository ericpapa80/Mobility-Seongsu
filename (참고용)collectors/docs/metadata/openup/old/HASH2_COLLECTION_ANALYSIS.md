# 성수동 전체 OpenUp 건물 데이터 수집 분석

## [hash2].txt 문서 분석 결과

### 확인된 사항

1. **새로운 access-token**
   - 토큰: `ff42c207-7967-40fe-89d9-5df14e0de026`
   - 이전 토큰(`295bae90-fbef-492f-9612-4b454f079680`)과 다름
   - 이전 토큰으로는 `401 Unauthorized` 오류 발생

2. **API 엔드포인트**
   - `/v2/pro/bd/hash`: cellTokens로 건물 목록 조회
   - 요청 형식: `{"cellTokens": ["357ca384"]}`
   - 응답 구조: `{"bd": {building_hash: building_info}, "keys": [...], ...}`

3. **cellToken 특성**
   - 하나의 cellToken으로 많은 건물 데이터 수집 가능
   - 예시: `357ca384` (강남구 신사동) → 수백 개 건물 반환
   - cellToken은 16진수 문자열 (8자리)

### 현재 상황

#### ✅ 구현 완료
- 새로운 access-token으로 API 클라이언트 초기화
- cellTokens로 건물 목록 수집 함수
- 건물 해시 키로 매장 목록 수집 함수
- 매장별 상세 매출 데이터 수집 함수

#### ⚠️ 필요한 사항

1. **성수동 전체를 커버하는 cellTokens 목록**
   - 현재 제공된 cellTokens: 13개
   - 이 cellTokens가 성수동 전체를 커버하는지 확인 필요
   - 성수동 영역을 그리드로 나누어 각 셀의 cellToken 생성 방법 필요

2. **cellToken 생성 방법**
   - 좌표 → cellToken 변환 알고리즘 불명확
   - OpenUp 내부 해시 함수 사용 가능성
   - 지도 그리드 시스템 기반일 가능성

3. **성수동 영역 정의**
   - 정확한 경계 좌표 필요
   - 행정동별 경계 정보 필요
   - 그리드 분할 크기 결정 필요

- **성공 시**: 성수동 건물 목록 수집 → 매장 ID 추출 → 매출 데이터 수집
