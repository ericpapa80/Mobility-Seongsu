# 병렬 처리 및 성수동 필터링 개선

## 개선 사항

### 1. 병렬 처리 지원
- **ThreadPoolExecutor**를 사용한 병렬 처리 구현
- 월별로 그룹화하여 월 내 날짜들을 병렬 처리
- API 호출 제한(초당 30 tps)을 고려한 스레드 수 조절

### 2. 성수동만 처리
- 성동구 전체 데이터 수집 후, 성수동 업소만 필터링
- 시계열 데이터 수집 시 성수동 업소 ID만 대상으로 처리
- 불필요한 데이터 수집 방지로 속도 향상

## 병렬 처리 구조

### 처리 방식
1. **월별 순차 처리**: 월 단위로 순차 처리하여 로그 및 저장 관리
2. **일별 병렬 처리**: 각 월 내 날짜들을 병렬로 처리
3. **스레드 안전성**: `Lock`을 사용하여 공유 데이터 구조 보호

### 설정
- `max_workers`: 기본값 5개 스레드
- `use_parallel`: 기본값 `True` (병렬 처리 활성화)

## 성능 개선 예상

### 병렬 처리 효과
- **이론적 속도**: 최대 5배 향상 (5개 스레드)
- **실제 속도**: API 호출 제한으로 인해 약 2-3배 향상 예상
- **네트워크 대기 시간**: 병렬 처리로 대기 시간 감소

### 성수동 필터링 효과
- **대상 업소 수**: 약 8,659개 (성동구 전체 17,205개 중)
- **필터링 비율**: 약 50% 감소
- **API 응답 크기**: 동일하지만 처리할 데이터만 감소

## 사용 방법

### 기본 사용 (병렬 처리 + 성수동만)
```bash
python scripts/sbiz/collect_seongsu_with_timeseries.py --timeseries-days 365
```

### 병렬 처리 비활성화
```python
collect_timeseries_data(
    start_date="20240101",
    end_date="20241231",
    target_bizes_ids=seongsu_bizes_ids,
    use_parallel=False  # 순차 처리
)
```

### 스레드 수 조절
```python
collect_timeseries_data(
    start_date="20240101",
    end_date="20241231",
    target_bizes_ids=seongsu_bizes_ids,
    max_workers=10  # 10개 스레드로 증가
)
```

## 주의사항

1. **API 호출 제한**
   - 초당 최대 30 tps
   - 스레드 수를 너무 많이 설정하면 API 제한에 걸릴 수 있음
   - 권장: 5-10개 스레드

2. **메모리 사용**
   - 병렬 처리 시 메모리 사용량 증가
   - 월별 중간 저장으로 메모리 관리

3. **에러 처리**
   - 개별 날짜 처리 실패 시에도 다른 날짜는 계속 처리
   - 에러 로그를 확인하여 재처리 가능
