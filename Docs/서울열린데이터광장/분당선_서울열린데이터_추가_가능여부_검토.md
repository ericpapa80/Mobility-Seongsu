# 분당선(서울숲역) 서울열린데이터 추가 가능 여부 검토

## 검토 대상

- **출처**: `서울시 지하철 호선별 역별 시간대별 승하차 인원 정보` (서울열린데이터광장)
- **추가 대상**: `public_20260225_성수뚝섬건대_시간대별_승하차_상세_카드권종별.csv`
- **추가 목적**: 분당선(수인분당선) 서울숲역 데이터 추가

---

## 1. 분당선 포함 여부 ✅ 가능

### API 호출 검증 결과

```
http://openapi.seoul.go.kr:8088/(인증키)/xml/CardSubwayTime/1/5/202111/분당선/서울숲/
```

| 호출 | 결과 |
|------|------|
| 202111/분당선/서울숲 | ✅ **정상** (HR_4~HR_23 GET_ON/GET_OFF 응답) |
| 202111/수인분당선/서울숲 | ❌ 해당하는 데이터 없음 |
| 202602/분당선/서울숲 | ❌ 해당하는 데이터 없음 (월별 갱신 대기) |

- **호선명**: `분당선` (수인분당선 아님)
- **역명**: `서울숲` (서울숲역 아님)

---

## 2. 구조 비교

### 대상 CSV (getStnPsgr 상세)

| 컬럼 | 단위 |
|------|------|
| date | 일별 (2026-02-25) |
| hour | 00~23 |
| stationName | 역명 |
| trnscdSeCdNm | 선불/후불/정기권/1회용 |
| trnscdUserSeCdNm | 일반/청소년/우대권 등 |
| rideNope, gffNope | 승차/하차 인원 |

### 서울열린데이터 (CardSubwayTime API)

| 항목 | 내용 |
|------|------|
| USE_MM | **월별** (YYYYMM) |
| SBWY_ROUT_LN_NM | 호선명 (분당선) |
| STTN | 역명 (서울숲) |
| HR_4_GET_ON_NOPE ~ HR_23_GET_OFF_NOPE | 04~23시 승·하차 |
| trnscd | **없음** (권종 미구분) |

---

## 3. 추가 시 고려사항

| 구분 | 대상 CSV | 서울열린데이터 |
|------|----------|----------------|
| **일/월** | 일별 (2026-02-25) | 월별 (202602) |
| **권종** | 선불/후불/청소년/우대권 등 | 없음 → `전체` 등으로 보완 |
| **시간** | hour 1열 | HR_x → hour로 변환 가능 |

- **2026-02-25**: USE_MM=202602 사용 시, 해당 월 데이터가 아직 없을 수 있음 (매월 5일 전월 갱신)
- **권종**: trnscdSeCdNm=`전체`, trnscdUserSeCdNm=`전체` 등으로 설정 후 추가 가능

## 4. 결론

| 질문 | 답변 |
|------|------|
| 분당선(서울숲역) 조회 가능? | ✅ **가능** (`분당선`/`서울숲`) |
| 기존 CSV에 추가 가능? | △ **가능** (형식 변환 필요) |
| 2026-02-25 일별 매칭? | △ 월별 데이터를 해당 일자 대용으로 사용 가능 |
| trnscd 상세? | × 권종 구분 없음 → `전체` 처리 |

## 5. API 호출 예시

```
http://openapi.seoul.go.kr:8088/(인증키)/xml/CardSubwayTime/1/5/202602/분당선/서울숲/
```

- **인증키**: 서울열린데이터광장에서 발급
- **USE_MM**: 202602 (2026년 2월), 202601 등
- **호선/역**: `분당선`, `서울숲` (URL 인코딩)

---

## 6. 수집 스크립트 연동 방향

1. 서울열린데이터 API로 `분당선/서울숲` 호출
2. HR_4~HR_23 GET_ON/GET_OFF → (hour, rideNope, gffNope) 행으로 변환
3. trnscdSeCd/trnscdUserSeCd = `전체` 등으로 채움
4. 기존 `public_20260225_...상세_카드권종별.csv`에 append 또는 별도 통합 CSV 생성
