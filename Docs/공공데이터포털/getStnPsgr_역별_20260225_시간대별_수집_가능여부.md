# Public_Data_api (getStnPsgr)를 통한 역별 시간대별 승하차인구 수집 가능 여부

## 요청 조건
- **역**: 성수역, 서울숲역, 뚝섬역, 건대입구역
- **날짜**: 2026-02-25
- **구간**: 시간대별
- **형식**: CSV

---

## 1. Public_Data_api (getStnPsgr) 현황

### API 정보
| 항목 | 내용 |
|------|------|
| Base URL | `https://apis.data.go.kr/B553766/psgr/getStnPsgr` |
| 방식 | GET |
| 파라미터 | `serviceKey`, `numOfRows`, `pasngYmd`, `stnNm`, `lineNm` |

### 지원 파라미터 (문서 기준)
- `pasngYmd`: 통행일자 (YYYYMMDD, 예: 20260225)
- `stnNm`: 역명 (URL 인코딩 필요, 예: 성수 → %EC%84%B1%EC%88%98)
- `lineNm`: 노선 (2=2호선, 7=7호선 등)

### 역별 lineNm 매핑
| 역명 | 노선 | lineNm |
|-----|------|--------|
| 성수역 | 2호선 | 2 |
| 뚝섬역 | 2호선 | 2 |
| 건대입구역 | 2호선 | 2 (2·7호선 환승 시 각각 조회 가능) |
| 서울숲역 | **수인분당선** | 별도 확인 필요 (예: 수인분당선 등) |

---

## 2. 시간대별 수집 가능 여부

### ⚠️ 제한사항
1. **스펙 불명확**: getStnPsgr 문서에 `시간대별` 파라미터가 없음
2. **응답 검증 불가**: 호출 시 500 Internal Server Error 발생 (날짜/서비스 이슈 가능)
3. **서울숲역**: 수인분당선 노선 코드가 lineNm과 어떻게 매핑되는지 문서에 없음
4. **데이터 제공 범위**: 과거/미래 날짜 지원 여부 미확인

### ✅ 이론적 가능성
- 공공데이터 역별승하차 API 중 일부는 **응답 내에 시간대별 필드**를 포함할 수 있음
- 서울교통공사_역별승하차인원(15143845) 등 다른 API는 `통행시간` 항목으로 시간대별 제공

---

## 3. 현재 프로젝트에서 가능한 방식 (참고)

프로젝트 내 **SK Open API (Puzzle)** 로는 이미 아래 조건의 **시간대별** 데이터를 확보해 두었음:

| 항목 | 내용 |
|------|------|
| 역 | 성수역(211), 뚝섬역(210), 서울숲역(K211), 건대입구역(212) |
| 날짜 | 2026-02-25 |
| 시간대 | 05~23시 (1시간 단위) |
| CSV | `Collect/raw/skopenapi_성수동_지하철역_승하차인구.csv` |

해당 CSV 필드: `datetime`, `date`, `hour`, `exit`(1=승차, 2=하차, 3=환승), `userCount` 등

## 4. 권장 사항

1. **getStnPsgr 정상 동작 확인**  
   - 최근 과거일(예: 어제)로 `pasngYmd` 지정 후 호출  
   - 응답 구조(특히 시간대 필드 유무) 확인

2. **대안 API 검토**  
   - [서울교통공사_역별승하차인원](https://www.data.go.kr/data/15143845/openapi.do) (15143845)  
   - 시간대별 데이터를 명시적으로 제공 (최근 7일 한정)

3. **서울숲역**  
   - getStnPsgr이 수인분당선을 지원하는지, lineNm 코드가 무엇인지 확인 필요

4. **CSV 변환**  
   - API 응답에 시간대 필드가 있다면, JSON → CSV 변환 스크립트로 바로 수집 가능
